## 🏗️ Architettura Sistema

### Layer Model (Database)
- **`Coa`**: Core certificate (serial, status, location, metadata)
- **`CoaSnapshot`**: Immutable traits snapshot
- **`CoaFile`**: PDF storage + integrity hash
- **`CoaSignature`**: Digital signatures (preparato per QES)
- **`CoaAnnex`**: Pro annexes (A_PROVENANCE, B_CONDITION, etc.)
- **`CoaEvent`**: Audit trail eventi

### Layer Services
- **`CoaIssueService`**: Emissione certificati con validazione location
- **`CoaPdfService`**: PDF generation (DomPDF + QR + watermark)
- **`VerifyPageService`**: Verifica pubblica certificati
- **`BundleService`**: Bundle PDF con annessi (Pro)
- **`HashingService`**: SHA-256 per integrity check
- **`SignatureService`**: QES interface (mock implementation)

### Layer Controllers
- **`CoaController`**: CRUD, emissione, PDF download, location update
- **`VerifyController`**: Verifica pubblica (rate-limited, no-auth)
- **`AnnexController`**: Gestione annessi Pro

## 📊 Validità Certificati

### Requisiti Validità (Short-Circuit)
1. **Status**: `valid` (non revoked)
2. **Traits Presence**: Almeno un trait significativo
3. **Author**: Determinabile da traits o user data
4. **Creation Date**: Anno/data creazione presente
5. **Location**: **NUOVO** - Luogo emissione obbligatorio

### Early-Exit Implementation
- Stop al primo requisito mancante
- UEM error codes con SweetAlert feedback
- Logging ULM per ogni failure path
